# ローカル会計アプリ (自治会会計システム) 引き継ぎ資料

このドキュメントは、これまでのスレッドで開発された「ローカル会計アプリ (`local-accounting-app`)」の現在の状態、実装済みの機能、および今後の開発に向けた課題（Next Steps）をまとめたものです。新しい開発スレッドに移行する際、AIアシスタントにこの内容をコンテキストとして共有してください。

## 1. プロジェクト概要
- **目的**: 任意団体（自治会・町内会など）向けの使いやすい会計・現出納帳Webアプリケーション
- **技術スタック**: Next.js (App Router), TypeScript, CSS Modules
- **バックエンド/DB**: microCMS (クラウドCMSコンソール)
- **ファイルストレージ**: Cloudflare R2 (S3互換・証憑等のアップロード用)

## 2. 完了している実装・フェーズ (これまでの進捗)
これまでにフェーズ1〜フェーズ3までの改修が完了しています。

### アーキテクチャとセキュリティ
- 前身の掲示板アプリの不要なデータ型コードをクリーンアップし、会計用構造に調整。
- **UI統合と認証**: 個別の「管理画面」と「ホーム画面（ゲスト用）」を統合し、1つのポータル画面（`/`）で完結するように改修しました。Basic認証を使ってアクセス制御を行い、未ログイン時は「閲覧モード（詳細等の閲覧・操作制限あり）」、Basic認証でログインした際は「管理モード（編集・追加などの全機能解放）」となる仕様です。

### データベース (microCMS)
以下のスキーマをAPI上で定義し、コード側（`src/lib/microcms.ts`, `src/types/index.ts`）でマッピングしています。
- **categories (勘定科目)**: id, 名前, タイプ (収入/支出), 順番 (order)
- **transactions (取引・収支データ)**: 日付, 勘定科目(リファレンス), 金額, **収入金額(`amount_income`)**, **支出金額(`amount_expense`)**, 詳細内容(`content`・リッチテキスト), 添付ファイル群(`attachments`・JSON配列)
  ※ 過去データ用のレガシーフィールド（`amount`, `receiptUrl`）に対するフォールバック処理も維持。

### フロントエンド / UI機能
1. **帳簿一覧・残高機能 (`/transactions`)**
   - 収入と支出で右寄せレイアウトを採用し、縦の区切り線で視認性を向上。
   - `fetchTransactions` とは別に、最古のデータから昇順に全取得してから**「差引残高」を自動計算**し、表示用に降順に並び直す処理 (`fetchAllLedgerTransactionsDesc`) を実装。
   - 各取引の行をマウスオーバーでハイライトし、行のどこをクリックしても取引詳細画面（`/transactions/[id]`）に遷移するUX。

2. **取引詳細画面 (`/transactions/[id]`)**
   - 取引情報の詳細を整理して表示（管理者のみ編集・削除ボタンが表示）。
   - 複数アップロードされたファイル（`attachments`）を表示。
   - `DocumentViewer` というカスタムコンポーネントを導入し、**画像とPDFのサムネイルプレビューとクリック時拡大表示（Lightbox表示）**を実現。

3. **入力・編集フォーム (`/transactions/new`, `/transactions/edit/[id]`)**
   - 数値入力フィールドのスピンボタン（上下矢印）をCSSで無効化。不要になった手動計算用の「残高」入力欄を削除。
   - 複数ファイルのドラッグアンドドロップアップロードと、ファイルごとのタイトル/説明プレビュー機能。
   - 編集画面にも `DocumentViewer` を組み込み、登録済み添付ファイルのサムネイルと「削除」ボタンを並記。
   - `content` に入力したテキスト内のURLを、保存直前に自動的にクリッカブルリンク(`<a>`タグ)に変換する自動リンク（linkify）処理を追加。

## 3. 次のステップ / 今後開発すべき課題 (Next Steps)
新しいスレッドでは、以下の項目の開発・改善から着手することが推奨されます。

1. **残高計算の取得上限（100件問題）の解決**
   - 現在の仕様では、一覧表示で残高を計算する際に `fetchAllLedgerTransactionsDesc` でmicroCMSから最大100件までを一括取得して計算しています。データが100件を超過した場合は正しい残高計算が崩れるため、microCMSのoffsetを利用した全件ループ取得、またはサーバー側（バッチ処理やWebhookフック）での残高計算値の直接保持など、本格的なページネーション/残高算出ロジックの導入が必要です。
2. **月別・年度別の絞り込み表示機能**
   - 帳簿一覧での「当月のみ表示」「指定月で絞り込み」などのフィルター機能UIの実装。
3. **レポート画面 (`/reports`) の拡充**
   - 勘定科目別での集計や、月ごとの収支グラフなど、より視覚的な分析画面の構築。
4. **決算書 / PDFエクスポート**
   - 監査などに提出できる形での出納帳や収支報告書の印刷用レイアウト/PDF出力機能。
5. **UIのモバイルレスポンシブ微調整**
   - 実機またはモバイル幅にした際の、帳簿一覧の横スクロールの挙動や各種ポップアップの最適化。
